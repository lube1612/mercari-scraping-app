# メルカリスクレイピング実行時の注意事項

## ⚠️ 停止してしまう場合の対処法

### 1. Google翻訳のポップアップが表示された場合

**手動で閉じる方法：**
- ポップアップの右上の「×」ボタンをクリック
- または、ポップアップ内の「英語」や「日本語」のタブをクリックして閉じる

**自動で閉じる処理：**
- コードに自動で閉じる処理を追加済みですが、うまく動作しない場合は手動で閉じてください

### 2. クッキー同意バナーが表示された場合

**手動でクリックする方法：**
- 「Got it」ボタンをクリック
- または「同意する」ボタンをクリック

**自動でクリックする処理：**
- コードに自動クリック処理を追加済みですが、うまく動作しない場合は手動でクリックしてください

### 3. 404エラーページが表示された場合

**自動処理：**
- 404エラーの場合は自動的にスキップして次の商品に進みます
- 手動で操作する必要はありません

### 4. スクリプトが完全に停止してしまった場合

**対処法：**
1. ターミナルで `Ctrl + C` を押してスクリプトを停止
2. ブラウザを閉じる
3. 再度 `python3 mercari/scrape.py` を実行

## 💡 推奨される実行方法

1. **スクリプトを実行**
   ```bash
   python3 mercari/scrape.py
   ```

2. **ブラウザが開いたら**
   - Google翻訳のポップアップがあれば「×」で閉じる
   - クッキー同意バナーがあれば「Got it」をクリック
   - その後は自動で処理が進みます

3. **404エラーの場合**
   - 自動的にスキップされるので、そのまま待っていてください

4. **正常に商品情報が取得できたら**
   - CSVファイルが `mercari/output/mercari_items.csv` に保存されます

## 🔧 トラブルシューティング

### 問題: ずっと停止している

**解決策：**
- ターミナルで `Ctrl + C` を押して停止
- ブラウザを閉じる
- 再度実行

### 問題: 商品情報が取得できない

**解決策：**
- ブラウザで実際にメルカリのページが正しく表示されているか確認
- ネットワーク接続を確認
- メルカリのサイトがメンテナンス中でないか確認

### 問題: CSVファイルが作成されない

**解決策：**
- ターミナルの出力を確認してエラーメッセージを確認
- `mercari/output/` フォルダが存在するか確認
