# プロジェクト構造ガイド

## 📁 推奨フォルダ構造

```
e2e/
├── common/                    # 共通モジュール
│   ├── __init__.py
│   ├── base_scraper.py        # ベーススクレイパークラス
│   └── utils.py               # 共通ユーティリティ
│
├── crowdworks/                # CrowdWorks用
│   ├── __init__.py
│   ├── scraper.py             # CrowdWorks専用スクレイパー
│   ├── scrape.py              # 実行スクリプト
│   ├── config.py              # 設定ファイル
│   └── output/                # 出力ファイル（CSVなど）
│       └── .gitkeep
│
├── mercari/                    # メルカリ用
│   ├── __init__.py
│   ├── scraper.py             # メルカリ専用スクレイパー
│   ├── scrape.py              # 実行スクリプト
│   ├── config.py              # 設定ファイル
│   └── output/                # 出力ファイル（CSVなど）
│       └── .gitkeep
│
├── dom_xpath_handler.py        # e2eモジュール（共通）
├── playwright_capture.py      # e2eモジュール（共通）
├── screenshot_processor.py     # e2eモジュール（共通）
├── e2e_test_runner.py         # e2eモジュール（共通）
│
└── README.md                   # プロジェクト全体の説明
```

## 🎯 この構造のメリット

1. **整理しやすい**: 各サイトのコードが分離されている
2. **拡張しやすい**: 新しいサイトを追加するのが簡単
3. **保守しやすい**: 各サイトの変更が他に影響しない
4. **再利用可能**: 共通機能を`common/`で共有

## 📝 使い方

### 新しいサイトを追加する場合

1. 新しいフォルダを作成（例: `mercari/`）
2. `common/base_scraper.py`を継承してスクレイパーを作成
3. 各サイトの特性に合わせて実装

### 共通機能を使う場合

```python
from common.base_scraper import BaseScraper
from common.utils import save_to_csv
```
